# 可以直接交给 Codex 的完整提示词

你是一个资深前端工程师 + 课堂互动产品设计师。请在当前项目中制作一个适合课堂 PPT 演讲的大屏互动网页小游戏，主题为：

《夺冠之路：女排精神中的马原密码》

## 一、项目定位

这个项目用于马克思主义基本原理课程展示。不要做复杂实时竞技游戏，而是做“实时投票 + 分支剧情 + 主持人推进”的课堂互动小游戏。

大屏页和主持人控制页必须是同一页。主持人可以在大屏页底部或右下角看到控制面板，也可以一键隐藏控制面板，以免影响投影展示。

## 二、页面要求

### 1. 大屏/主持人同页

路径：

```text
/screen
```

如果路由不方便，首页也可以直接显示大屏页。

大屏页需要包含：

- 游戏标题
- 当前剧情
- 当前问题
- A/B/C/D 四个选项
- 每个选项的实时票数
- 每个选项的百分比进度条
- 当前阶段状态：等待开始 / 剧情导入 / 投票中 / 结果锁定 / 最终总结
- 二维码，指向手机投票页
- 主持人控制面板

主持人控制面板需要有：

- 开始
- 开放投票
- 锁定结果
- 按多数推进
- 重置
- 隐藏/显示控制区
- 手动给 A/B/C/D 加减票的按钮，用于现场网络异常或演示补救

### 2. 手机投票页

路径：

```text
/vote
```

手机端只需要显示：

- 当前轮次
- 当前问题
- A/B/C/D 选项
- 提交状态
- 如果主持人未开放投票，则按钮禁用并提示等待

要求：

- 同一设备在同一轮只能算一票，但投票锁定前可以改选。
- 改选时应从原选项扣一票，再给新选项加一票。

## 三、实时逻辑

优先使用：

- React + Vite
- Express
- Socket.IO

不要依赖 Firebase 或 Supabase，这样课堂展示时不需要注册外部账号。

运行逻辑：

- 开发模式：`npm run dev`
- 生产模式：`npm run build` 后 `npm start`
- 大屏与手机在同一 Wi-Fi 下，通过局域网 IP 访问
- 大屏页显示二维码，手机扫码进入 `/vote`

## 四、剧情结构

总共四幕：

1. 第一幕：0∶2 之后的暂停
2. 第二幕：找到主要矛盾
3. 第三幕：精神如何变成力量
4. 第四幕：赛点辩论

每幕都是 A/B/C/D 四个选择。

第一幕决定主路线：

- A：信念牵引路线
- B：集体协作路线
- C：逆境转化路线
- D：实践创新路线

后续每轮仍然投票，但剧情旁白要根据第一幕的主路线轻微变化，使其看起来像分支剧情，而不是普通问卷。

最后根据第一幕主路线生成总结页，同时列出全班每一轮选择链。

## 五、理论内容要求

必须自然结合马克思主义基本原理，不要机械堆概念。重点使用这些概念：

- 物质与意识的辩证关系
- 意识的能动作用
- 实践是认识的基础
- 实践是检验真理的标准
- 一切从实际出发，具体问题具体分析
- 主要矛盾
- 矛盾主要方面
- 矛盾双方在一定条件下相互转化
- 量变与质变
- 普遍联系
- 整体与部分
- 个人与社会
- 人民群众是历史创造者
- 事物发展是前进性与曲折性的统一

女排精神关键词：

- 祖国至上
- 团结协作
- 顽强拼搏
- 永不言败

不要把“女排精神”和“马原概念”硬凑；要通过比赛情境、战术选择、集体配合、临场调整来体现理论。

## 六、UI 风格

UI 要和 PPT 保持一致，适合大屏投影。

视觉关键词：

- 红金主色
- 深蓝背景
- 庄重但有比赛感
- 字体大
- 选项清楚
- 票数醒目
- 按钮少但明确
- PPT 汇报风，不要做成花哨手游

推荐配色：

```text
深红：#8B0000
金色：#F5C542
深蓝：#102A43
米白：#FFF8E7
```

布局：

- 顶部：标题 + 当前阶段
- 左侧：剧情 + 问题 + 主持人讲解提示
- 右侧：选项票数卡片
- 底部：主持人控制区，可隐藏
- 手机端：单列卡片式

## 七、代码结构建议

```text
src/
  App.jsx
  gameData.js
  styles.css
server.js
package.json
vite.config.js
README.md
```

`gameData.js` 中集中放：

- 游戏标题
- 每一幕剧情
- 每一幕问题
- A/B/C/D 选项
- 每个选项的马原解释
- 每个选项的结果文本
- 最终路线总结

这样非程序同学也能直接改文案。

## 八、质量要求

1. 代码必须能运行。
2. 不要引入过重框架。
3. 不要把主持人页拆成单独页面。
4. Socket.IO 断开时，页面要至少显示连接状态。
5. 平票默认按 A/B/C/D 顺序处理，但要保留主持人手动加减票。
6. 控制区按钮状态要合理，比如没开始时不能锁定，没锁定时不能推进。
7. 移动端投票页要适配手机。
8. 大屏端字体要足够大，适合教室投影。
9. README 里写清楚如何运行、如何扫码、如何修改剧情。
10. 请完成后检查是否有明显运行错误，并说明你改了哪些文件。

## 九、可直接采用的四幕剧情

请使用或参考 `src/gameData.js` / `STORY_BOARD.md` 中的剧情文案，不要重新写成普通选择题。
